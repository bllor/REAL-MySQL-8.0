2.1.서버 설정

1.시스템 변수
설정 파일의 내용을 읽어 메모리나 작동 방식을 초기화하고, 접속된 사용자를 제어하기 위한 별도의 값

[1]시스템 변수의 속성
{1}cmd-line
서버의 명령행 인자로 설정될 수 있다는 뜻
{2}option file
MySQL의 설정파일 my.cnf로 제어할 수 있는지 여부를 나타냄
{3}system var
시스템 변수인지 아닌지 나타냄
_를 이용하여 나타내는 것으로 변경되는 중
{4}var scope
시스템 변수의 적용범위를 나타냄
global : mysql 서버 전체
session : 서버와 클라이언트 간의 커넥션만 범위인지
both : 둘다 해당되는지
{5}dynamic
시스템 변수가 동적변수인지 정적변수인지 나타낸다.

[2]글로벌 변수와 세션 변수
글로벌 변수 : 하나의 MYSQL 서버 인스턴스에서 전체적으로 영햐응ㄹ 미치는 시스템 변수
시스템 변수 : mysql클라이언트가 mysql 서버에 접속할 때 기본으로 부여하는 옵션의 기본값을 제어하는데 사용.

[3]정적변수와 동적변수
동적 변수 : 서버가 기동 중인 상태에서 변경이 가능
정적 변수 : 서버가 기동 중일 때 변경이 불가능

MySQL 5.7과 8.0의 차이점

1.사용자 인증방식 변경
mysql 5.7에서는 native authentication인증방식을 사용하지만,
mysql 8.0에서는 caching sha-2 authentication인증방식을 사용
만약 mysql 8.0에서 native authentication인증방식을 사용하려고 한다면
--default-authentication-plugin=mysql_native_password 파라미터를 사용

2.외래키 이름의 길이
mysql 8.0에서는 외래키의 이름이 64글자로 제한된다.

3.파티션을 위한 공용 테이블스페이스 저장여부
mysql 8.x에서는 파티션의 각 테이블스페이스를 고용 테이블스페이스에 저장할 수 없다.

*
mysql 8.0에 새롭게 추가된 기능

1.별도의 NoSQL 문서 데이터베이스가 필요하지 않다.
2.SQL Locking절에 no wait 및 skip loaded대안을 제공한다.
3.인덱스를 내림차순으로 지원한다.
4.json 경로 표현식의 범위에 대한 구문을 확장한다.
